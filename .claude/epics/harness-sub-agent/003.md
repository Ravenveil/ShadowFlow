# Task: 关卡型角色模板

## Description

角色模板库新增 3 个 "驳回 / 放行" 专职模板，让用户搭团队时有现成关卡角色可选，不用从零写 soul。

## Acceptance Criteria

- [ ] `shadowflow/runtime/role_templates/` 目录建立（如已有 catalog_service 复用）
- [ ] 三个种子模板（每个含完整 RoleProfile yaml）：
  - **Gatekeeper** — 在编码开始前校验需求清晰度 / 设计可行性 / 集成风险；可驳回到需求或设计角色
  - **Code Reviewer** — 验证规范对齐 / 设计还原 / 隐藏缺陷；可驳回到 Dev
  - **QA Tester** — 验证功能正确性 / 边界 / 回归 / 稳定性；可驳回到 Dev 或上游
- [ ] 每个模板填充完整深度契约字段（responsibilities / constraints / handoff_rules）
- [ ] Quick Hire 弹层加 "从模板招"路径，能从 catalog 一键 instantiate 到当前 team
- [ ] 文档化每个模板用法 + 出处（参考文章 §1.3）

## Technical Details

- 模板格式参考 BMAD-METHOD agent 文件结构
- handoff_rules 字段示例参考文章中 Gatekeeper 的驳回路径
