# Task: Team dev-map / Task Board 模板

## Description

对应 §4 B6 —— 给每个 team 自动生成两个项目记忆 artifact：`dev-map.md`（代码 / 模块结构入口）
和 `task-board.md`（需求 / 任务历史入口），由角色契约规定谁更新。

## Acceptance Criteria

- [ ] team 创建时（无论是自然语言生成还是 quick hire）自动初始化两个 markdown artifact：
  - `dev-map.md` —— 含模板章节："本团队管理的模块入口" / "标准实现模式" / "修改 X 模块的影响范围"
  - `task-board.md` —— 含模板章节："进行中" / "已完成" / "blocker"
- [ ] artifact 存到 team workspace 目录（与现有 artifact 同位置）
- [ ] [[harness-sub-agent]] 003 Code Reviewer / Dev / PM 等模板默认 constraint："修改代码必更新
      dev-map" / "推进任务必更新 task-board"
- [ ] Runtime 在 turn 结束时检查这两份文件是否被修改（若 agent 改了相关模块但没更新 dev-map，
      发 telemetry warning）

## Technical Details

- 参考文章 §1.8 "dev-map + Task Board" 原文
- 与 [[harness-scripts]] Validation Hook 联动：可以做一个 builtin validator "dev-map-fresh"
