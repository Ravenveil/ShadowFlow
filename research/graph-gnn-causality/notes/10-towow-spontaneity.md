# Towow 与自发协作

> 目的：提炼 `ToWow` 对 `ShadowFlow` 自发层的启发

## 一句话结论

`ToWow` 的价值不在于它给了我们一个现成 workflow，而在于它提醒我们：

- 参与者不一定预先已知
- 结构不一定预先已知
- 子需求可能在协作中递归长出来

## 对我们最有启发的点

1. `goal formulation`
2. `participant matching`
3. `negotiation before execution`
4. `create_sub_demand`
5. “不要预设重要性，让重要性从共振中涌现”

## 这些点怎么翻译到 ShadowFlow

### Towow 语言

- 共振
- 协商
- 结晶
- 子需求

### ShadowFlow 更工程化的语言

- relevance / fit
- assembly draft / mutation
- compile / validate
- child run / delegated subgoal

## 最值得接上的位置

不是 runtime 替换，而是：

1. assembly 前的 participant / role activation
2. assembly 中的 delegate / create_subgoal
3. runtime 后的 feedback 回流

## 风险

如果我们直接抄 ToWow 的叙事，而不先把 assembly 本体做稳，
就很容易得到：

- 很强的产品想象
- 很弱的可执行结构

所以 Towow 最适合成为：

- 我们的上层灵感
- 我们的协作哲学参考

而不是当前的主引擎替身。
