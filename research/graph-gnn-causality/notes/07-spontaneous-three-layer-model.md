# 三层模型说明

> 目的：把“积木本体 / LLM 装配助理 / 自发协作层”讲清楚

## 第一层：积木本体

这是系统的硬地板。

它定义：

- block 是什么
- block 怎么连
- 哪些连法合法
- 怎样编译成可执行 workflow

没有这层，就没有真正的 assembly。

## 第二层：LLM 装配助理

这是系统的大脑接口层。

它不负责执行 run，也不直接学习最优策略。

它负责：

- 把目标翻译成 assembly draft
- 解释约束错误
- 修理不合法组合
- 提供建议，但不越过 constraint

它的关键词不是“推荐模板”，而是：

- draft
- critique
- repair
- mutate

## 第三层：自发协作层

这是让系统开始“自己更会长”的那一层。

它利用：

- 反馈
- 图结构
- 历史结果
- 学习策略

去逐步调整：

- 选哪些 block
- 连哪些边
- 派谁做子任务
- 什么时候该 review / retry / split

## 为什么三层不能直接跳过第二层

因为如果没有第二层：

- 人直接面对 block catalog 太重
- 第三层拿不到结构化起点
- 自由装配要么退化成手工拖图，要么退化成自然语言乱生成

所以第二层是桥。
