## A · Markdown chip 提取实现复核

**Result: PASS**.

- `extractRows.ts:48` — `FENCED_BLOCK_RE = /```([^\s`\n]*)([^\n]*)\n([\s\S]*?)```/g`
  uses non-greedy capture so consecutive fences don't merge. Captures lang +
  info-string + body separately.
- `BASH_LANGS` set covers `bash / sh / shell / zsh / console / cmd / powershell /
  ps / ps1` (line 37) — fenced blocks in those langs are routed to `bash-chip`,
  others to `code-chip`. Sensible coverage.
- `DOLLAR_LINE_RE = /^[ \t]*\$ (.+)$/gm` matches inline `$ cmd` lines outside
  fences. Restricted to `$ ` (dollar + space) avoids jQuery/regex false positives.
- `HEADING_LINE_RE = /^[ \t]*(#{2,3})[ \t]+(.+?)...$/gm` extracts `##` and `###`
  ATX headings (intentionally skips `#` to avoid grabbing the LLM's first
  emphasized line). Reasonable scope.
- Algorithm correctness: outer loop pulls fenced blocks; `scanProseRange()` does
  two-pass token collection (headings + dollar lines), sorts by index, walks
  in document order. Handles consecutive fences (no spurious empty text rows —
  verified by test "does not emit spurious empty text rows between adjacent
  fences").
- 9 unit tests in `extractRows.test.ts` cover: pure prose, empty/whitespace,
  fenced bash+surrounding prose, mixed-lang fences in order, inline `$` lines,
  consecutive fences, `sh/shell/zsh` aliases, `##/###` headings, `title="..."`
  info-string label extraction.
- 5 integration tests in `AssistantText.render.test.tsx` verify the DOM actually
  renders distinct chip elements, not one opaque pre block.

Visual chip-vs-bubble check (`timeline.module.css:806-820`):
- `.bashChip` / `.codeBlock`: `border-radius: 6px`, `border: 1px solid
  var(--border-subtle)`, `background: var(--bg-elev-1)`. 6px radius + 1px hairline
  border is **chip** territory, not bubble (bubble = ≥10px radius + filled accent
  bg, per the user's TRAE expectation).
- `.chipHeader`: `background: transparent`, `padding: 5px 10px` — flat
  inline header.
- `.chipBadge`: 3px radius — micro-pill, acceptable.
- `.chipExpanded`: `background: var(--bg)` mono recess, `border-top: 1px solid`
  — the canonical "code block recess" pattern.

**Verdict: chip styling is on-spec for TRAE/OpenDesign flat-chip aesthetic.**
