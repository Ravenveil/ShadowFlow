# 🌀 Shadow 体系战略：数字逻辑与具身执行的统一编排

## 1. 体系架构：数字与物理的智能化层级

我们正在构建一个从“信息处理”到“物理操纵”的完整闭环：

*   **Shadow (知识中心与记忆管理员 - The Librarian)**: 
    - 负责所有技术文档、API 手册的索引。
    - 管理机器人的“经验缓冲区（Experience Buffer）”，存储成功与失败的快照。
*   **AgentGraph (统一编排大脑 - The Orchestrator)**: 
    - **方向一：代码工作流**。负责复杂逻辑推理、代码生成、自动化测试与 Debug（类 Claude 模式）。
    - **方向二：具身工作流**。源自 OpenClaw，将 ShadowClaw 的物理能力序列化为可编排的积木（类 Robot Control 模式）。
    - **核心引擎**：集成 LangGraph，通过“状态机”管理逻辑/动作的循环与自愈。
*   **ShadowClaw (物理执行器 - The Effector)**: 
    - 安装在机器人上的 Rust 核心，提供高并发、实时响应的底层控制。
    - 将传感器数据（力矩、坐标）实时反馈给 AgentGraph 的状态对象。
*   **ai-sql-agent (闭环逻辑 DNA - The DNA)**: 
    - 提供成熟的“执行-反馈-纠错”回路模版，通用于代码修复与动作修正。

## 2. 自动化搭积木的“双模”机制

### 模式 A：代码工作流 (Code Flow)
- **目标**：实现“意图 -> 代码设计 -> 编写 -> 验证 -> 修正”的自动化。
- **原子块**：代码读写、单元测试运行器、静态分析工具。

### 模式 B：具身工作流 (Physical Flow / OpenClaw)
- **目标**：实现“意图 -> 环境感知 -> 路径规划 -> 动作执行 -> 结果确认”的自动化。
- **原子块**：视觉定位、轨迹规划、力控抓取、避障。

---

## 3. 规范化原子节点 (Atomic Block Specification)

所有节点（无论数字或物理）必须包含以下描述以支持自动化编排：

- **Identity**: 全局唯一标识。
- **Domain**: `Code` 或 `Physical`。
- **Metadata**: 自然语言功能描述（供 Planner 匹配）。
- **Interface**: 
    - **Input Schema**: 输入参数的 JSON 格式。
    - **Output Schema**: 执行结果的 JSON 格式。
- **State Impact**: 该节点执行后会对全局 `State` 产生何种改变。

---

## 4. LangGraph：AgentGraph 的核心优势

集成 LangGraph 是为了解决“一次性成功率”低的难题：
- **循环纠错 (Loops)**：代码运行失败自动跳回修复；抓取滑落自动跳回重定位。
- **状态持久化 (Checkpoints)**：支持跨会话的任务恢复，这对于长周期的代码重构或长距离的机器人任务至关重要。
- **断点确认 (Human-in-the-loop)**：在关键节点挂起任务，等待用户在 Shadow 界面确认。

## 5. 执行路线图

### 第一阶段：元数据驱动的“积木库”标准化
- 在 Shadow 中建立统一的积木元数据索引，涵盖代码工具与机器人技能。

### 第二阶段：AgentGraph 状态机重构 (基于 LangGraph)
- 参考 `ai-sql-agent`，设计支持双模（代码/物理）的状态机结构。
- 实现 `LLMPlanner`，支持根据用户意图自动检索积木并连线。

### 第三阶段：ShadowClaw & OpenClaw 原子切片
- 将机器人底层的 Rust 逻辑封装为符合标准的原子节点，支持实时状态反馈。

## 6. 最终愿景：全场景智能

未来的 Shadow AI 能够根据指令自由切换维度：在数字世界里重构代码，在物理世界中操纵物体。所有的逻辑编排、记忆沉淀与执行反馈都统一在这一套体系之中。
