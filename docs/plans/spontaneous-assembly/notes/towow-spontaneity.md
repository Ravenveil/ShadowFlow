# Towow 与自发协作

> 目的：提炼 `ToWow` 对 `ShadowFlow` 自发层的启发

## 一句话结论

`ToWow` 的价值不在于它给了我们一个现成 workflow，而在于它提醒我们：

- 参与者不一定预先已知
- 结构不一定预先已知
- 子需求可能在协作中递归长出来

## 对我们最有启发的点

1. `goal formulation`
2. `participant matching`
3. `negotiation before execution`
4. `create_sub_demand`
5. “不要预设重要性，让重要性从共振中涌现”

## 最值得接上的位置

不是 runtime 替换，而是：

1. assembly 前的 participant / role activation
2. assembly 中的 delegate / create_subgoal
3. runtime 后的 feedback 回流
