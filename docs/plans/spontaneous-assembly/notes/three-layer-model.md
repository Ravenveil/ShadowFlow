# 三层模型说明

> 目的：把“积木本体 / 局部激活层 / 自发协作层”讲清楚

## 第一层：积木本体

这是系统的硬地板。

它定义：

- block 是什么
- block 怎么连
- 哪些连法合法
- 怎样编译成可执行 workflow

没有这层，就没有真正的 assembly。

## 第二层：局部激活层

这是系统的局部选择层。

它不负责全局拍板，也不直接学习最优策略。

它负责：

- 激活和当前目标最相关的 role / block / delegate 候选
- 缩小动作空间
- 识别 gap 和 subgoal 触发点
- 为后面的学习层准备可控候选集

它的关键词不是“中央规划”，而是：

- activate
- gate
- select
- trigger

## 第三层：自发协作层

这是让系统开始“自己更会长”的那一层。

它利用：

- 反馈
- 图结构
- 历史结果
- 学习策略

去逐步调整：

- 选哪些 block
- 连哪些边
- 派谁做子任务
- 什么时候该 review / retry / split

## 为什么三层不能直接跳过第二层

因为如果没有第二层：

- 人直接面对 block catalog 太重
- 第三层拿不到局部候选空间
- 系统要么退化成手工装配，要么退化成中央 LLM 乱拍板

所以第二层是桥，而且这个桥更应该先落到执行层的 gating 语义里。
