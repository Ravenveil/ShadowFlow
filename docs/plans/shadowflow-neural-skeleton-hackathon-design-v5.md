## 1. v5 相对 v4 的核心变化（一句话）

**把 v4 "对话式装配 workflow 引擎" 改为 "神经元式积木激活骨架 + 训练闭环通电"**。不再追求 demo 可观性，不和 Coze/n8n 抢通用场景，聚焦把半年前已经设计好的 `ActivationBandit`（激活学习器）接入主管道，扩充积木库到足以看到"神经元式激活"涌现。

User 原话：**"我们团队就是每一个都是一个积木块。然后这个积木块，它是可以像神经元一样去自由的去激活，然后通过这样的训练，然后去搭建一个成熟的智能体。"**

## 2. 问题重述

ShadowFlow Phase 0-2 已经定义了完整的神经元式装配系统骨架：
- 积木块 = `WorkflowBlockSpec` + `SkillSpec` + `RoleSpec`
- 激活器 = `ActivationSelector`（贪心版已做）
- 学习器 = `ActivationBandit`（Phase 3 Step 1 已做）
- 拓扑推断 = `ConnectionResolver` v2（capability 依赖版已做）
- 约束 = `AssemblyConstraintSpec`
- 反馈 = `ExecutionFeedbackRecord` + `ActivationTrainingSample`

**但 `ActivationBandit` 是孤岛**。它存在，测试通过，但没有和 `RuntimeService` / `ActivationSelector` / CLI 连起来。**没有完整的训练闭环**。

**黑客松 4 周的核心工程任务**：通电。加三条关键电线，扩充积木库到 25+ 个，跑 50-80 次真实任务产生训练数据，看神经元激活收敛。

## 5. 分阶段执行清单（Day 级）

### 阶段 1: 前置调研（Day 1-2，Week 1 开头）

**Day 1**
- 上午：克隆 n8n 源码，读 `packages/nodes-base/nodes/` 下 5 个典型节点（Gmail / HttpRequest / Code / Webhook / Cron）
- 下午：克隆 Coze Studio 源码，读工作流引擎核心、节点定义
- **P11 硬验证**：同步做——0G testnet 钱包创建、faucet 抽满、indexer 上传一次"hello world"文件

**Day 2**
- 上午：写 `docs/plans/block-spec-comparison-v1.md`，对比 n8n / Coze / ShadowFlow 的积木抽象
- 下午：基于对比决定是否改 `WorkflowBlockSpec` schema，如果要改，改 `shadowflow/highlevel.py`
- **P11 硬验证继续**：hackquest 提交格式确认、license 决定（默认 MIT）、hosting 决定（默认 Fly.io 免费层 + 录屏备份）

### 阶段 2: 激活学习器通电（Day 3-7，Week 1 剩余）

**Day 3**：第一条电线——运行结束后自动导出训练数据
- 文件：`shadowflow/runtime/service.py` 的 `run_workflow` 方法
- 加 hook：`try: ... finally: dataset = export_activation_training_dataset(run_id); _append_to_training_store(dataset)`
- 训练 store 路径：`~/.shadowflow/training/<slug>.jsonl`
- 测试：跑一个简单 workflow，确认 jsonl 生成

**Day 4**：第二条电线——装配上下文知道学习器的存在
- 文件：`shadowflow/assembly/activation.py` / `shadowflow/highlevel.py`
- `AssembleContext` 加字段 `learner: Optional[ActivationBandit] = None`
- `ActivationSelector.select()` 加参数 `bandit: Optional[ActivationBandit] = None`
- 单元测试覆盖三种情况：bandit 为 None、bandit 样本不够、bandit 样本够了

**Day 5**：第三条电线——bandit 真的影响选择
- 文件：`shadowflow/assembly/activation.py` 的 `select()` 方法
- 动作：`if bandit and bandit.is_trained(): scored_blocks = bandit.score(goal, candidates); use those weights; else: fall back to greedy`
- 测试：手动造训练数据，验证训练前后 select 结果变化

**Day 6**：CLI 参数暴露
- 文件：`shadowflow/cli/` 下的 `do` 命令
- 加 `--learn` / `--no-learn` 参数，默认 `--learn`
- 文档：`docs/CLI_INSTALL_AND_USAGE.md` 加说明

**Day 7**：端到端收敛演示脚本
- 写 `scripts/demo_bandit_convergence.py`
- 动作：跑同一类任务 15 次，每次记录激活的积木集合，输出 ASCII 对比图

**Week 1 末检查点**：跑 `python scripts/demo_bandit_convergence.py`，看到激活从均匀到稀疏的过渡。**这一步没做出来，Week 2-4 全部推迟**。

### 阶段 3: 积木库扩充（Day 8-14，Week 2）

**Day 8-9：输入类积木（触发器）**
- `TriggerCronBlock`（定时触发）：`croniter` 库
- `TriggerWebhookBlock`（Webhook 触发）：FastAPI 端点暴露

**Day 10-11：通用能力积木**
- `HttpRequestBlock`：`httpx` 封装 + 4 种认证
- `CodeExecBlock`：Python subprocess 隔离（不追求完美沙箱）
- `ConditionalBlock`、`AggregateBlock`

**Day 12-13：输出类积木**
- `NotifyLarkBlock`、`NotifySlackBlock`、`NotifyDiscordBlock`
- `FileWriteBlock`、`FileReadBlock`

**Day 14：智能类积木**
- `LlmChatBlock`、`LlmSummarizeBlock`：复用 Phase 0-1 provider adapter
- `EmbeddingBlock`、`RssFetchBlock`、`GitHubSearchBlock`

**Week 2 末检查点**：`shadowflow registry list --kind blocks` 显示 25+ 个积木。**如果 Day 14 提前完成，可选进入 P12 B 包迷你版扩展（写 5-10 个代码团队自治积木）**。

### 阶段 4: 场景装载 + 训练（Day 15-21，Week 3）

**Day 15**：定义场景装载配置
- 新建 `configs/scenarios/online_automation.yaml`
- 声明积木白名单

**Day 16-18**：手动跑 50-80 次任务产生训练数据
- 写 50-80 条线上自动化任务文本（可让 AI 批量生成）
- 每条跑 `shadowflow do --goal "<任务>" --scenario online_automation`

**Day 19-20**：观察收敛
- 跑 `demo_bandit_convergence.py`，对比前后激活模式
- 诊断收敛问题（如果有）

**Day 21**：训练数据同步 0G Storage
- 复用 `ZeroGCheckpointStore`
- **可选**：如 P12 B 包已做，这里用 B 包再跑 15-20 次训练数据，证明可插拔

**Week 3 末检查点**：训练数据集有 50+ 条样本，激活模式可见收敛。

### 阶段 5: 打磨 + 提交（Day 22-28，Week 4）

**Day 22-23**：演示剧本 + CLI 输出打磨
**Day 24-25**：端到端联调、修 bug
**Day 26**：README 提交版
**Day 27**：录 3-5 分钟演示视频（可选）
**Day 28**：提交前 sanity check

## 9. 一句话 pitch

> ShadowFlow 是 **神经元式积木激活骨架**。你装载一批积木（触发器 + HTTP + 代码 + 大模型 + 通知 + 自定义），说一句话目标，系统动态激活相关积木、自动推断依赖拓扑、执行、从反馈中学习。跑得越多，激活越精准，最终收敛成一个"成熟的智能体"。**骨架不变，换一套积木就是换一个应用**——Week 3 装线上自动化积木，post-hackathon 换代码团队自治积木（对标 towow 6 层机制），更远期换 embodied 家居机器人积木。

## 尾注：我三次偏离主线的教训

user 半年前已经把 ShadowFlow 的灵魂（神经元式积木激活 + 训练收敛）设计好写在 `roadmap.md` Phase 3 Step 1 里。我这几小时做的"三次 pivot"每一次都在抹掉这个灵魂换一个"更容易解释给评委"的东西。正确姿势：**信任 user 半年前的设计，把已经写好的组件真正通电**。

这份 v5 就是回到主线。不再提 bootstrap、不再提规则驱动、不再提 meta-workflow。就是 Phase 3 Step 1 的 ActivationBandit 完整 wire-up + 积木库扩充 + 真实训练数据累积。
