## 设计理念

**记忆应该像河流一样流动，而不是静止的池子。**

- 🌊 **流动性**：记忆持续流动，不是静态存储
- 🔄 **汇聚与分流**：节点产出汇入主流，节点需求从主流分流
- 📍 **位置感**：上游产出，下游可获取；越下游记忆越丰富
- 🏝️ **沉淀**：重要记忆可"沉淀"在河床，长期保留
- 🌊 **自由存取**：节点可以自由取水（读）、自由注水（写）

## 四层融合设计

### 🏔️ Layer 1: 源头（Claude Code 分级记忆）

```
┌─────────────────────────────────────────────────────────────────────────┐
│  workflow.md     → 项目架构、全局规则（最高优先级）                        │
│  .workflow/rules/ → 条件规则，按节点类型匹配加载                           │
│  类似 paths 字段：globs: ["execution/code/**"]                           │
└─────────────────────────────────────────────────────────────────────────┘
```

### 🌊 Layer 2: 主流（实时流动的记忆）

| 记忆类型 | 用途 | 示例 |
|----------|------|------|
| **context** | 上下文：任务理解、决策依据 | 设计决策、需求分析 |
| **execution** | 执行结果：节点产出 | 代码、文档、数据 |
| **working** | 工作记忆：临时状态 | 计数器、标志位、循环变量 |
| **knowledge** | 知识记忆：持久化知识 | 模式、规则、经验 |

**特点**：自由流动，节点可随时取用或汇入

### 🏝️ Layer 3: 沉淀层（Auto Memory + 时序图谱）

**自动沉淀条件**：
- 用户纠正过的输出
- 成功模式（出现>3次）
- 失败后找到的解决方案

**时序图谱（Zep）**：
- 每个记忆带时间戳
- 实体关系建模
- 支持时间范围查询

**入口限制**：前200行自动加载，其余按需读取

### 🚧 Layer 4: 水闸（LangGraph 检查点）

**创建时机**：
- 节点完成后
- 条件边评估前
- 工作流暂停时

**检查点内容**：
- 完整水位快照（记忆池状态）
- 当前位置（执行到哪个节点）
- 执行路径

**能力**：
- 从任意水闸恢复
- 时间旅行调试
- 审计追踪

## 节点与河流的交互接口

```typescript
interface RiverMemoryAccess {
  // ===== 取水（读取）=====

  /** 从主流取水 - 自由读取 */
  drink(type?: MemoryType): IMemoryChunk[];

  /** 用过滤网取水 - 条件查询 */
  scoop(filter: {
    type?: MemoryType;
    sourceNode?: string;
    timeRange?: { from: Date; to: Date };
  }): IMemoryChunk[];

  /** 从沉淀层取水 - 读取学习到的模式 */
  dredge(): IPattern[];

  // ===== 注水（写入）=====

  /** 向主流注水 - 自由写入 */
  pour(chunk: IMemoryChunk): void;

  /** 向沉淀层注水 - 记录学习 */
  settle(pattern: IPattern): void;

  // ===== 水闸操作 =====

  /** 建闸 - 创建检查点 */
  buildDam(): string;  // 返回检查点ID

  /** 开闸 - 恢复到检查点 */
  openDam(checkpointId: string): void;

  /** 查看所有水闸 */
  listDams(): ICheckpoint[];
}
```

## 节点使用示例

```typescript
// Code 节点
class CodeNode {
  async execute(context: INodeContext) {
    const river = context.memory; // 河流访问接口

    // 1. 取水：获取上下文记忆
    const contextMemory = river.drink('context');
    const decisions = river.scoop({
      sourceNode: 'analyze',
      type: 'execution'
    });

    // 2. 执行：生成代码
    const code = await this.generateCode(contextMemory, decisions);

    // 3. 注水：汇入主流
    river.pour({
      type: 'execution',
      content: code,
      metadata: { importance: 0.8 }
    });

    // 4. 沉淀：如果发现新模式
    if (this.isNewPattern(code)) {
      river.settle({
        type: 'code_pattern',
        content: code.pattern,
        reason: '用户认可的代码风格'
      });
    }
  }
}
```
