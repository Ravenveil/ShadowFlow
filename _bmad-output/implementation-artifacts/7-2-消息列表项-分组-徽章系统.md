# Story 7.2: 消息列表项 + 分组 + 徽章系统

Status: ready-for-dev
Created: 2026-04-21T07:05:42Z

---

## Story

As a **用户**,
I want **列表项一眼看出哪个群在跑、有几条待审议、哪个员工在等我**,
so that **我不用逐个打开就知道今天的优先级**。

---

## Acceptance Criteria

### AC1: 分组 section header

**Given** `MessageList` 骨架已由 Story 7.1 建立,内容区有 `TEAM RUNS` / `AGENT DMs` 两段占位
**When** 数据加载完毕
**Then** 列表按两段分组渲染:
- `TEAM RUNS` section → 群聊条目(groupRoster)
- `AGENT DMs` section → 单聊条目(agentRoster DM)
**And** 每个 section header:10px `font-mono text-white/50 uppercase tracking-[0.1em]`
**And** 若某 section 无数据,section header 仍显示但下方空态"暂无会话"

### AC2: 消息列表项(MessageItem)

**Given** 数据从 `GET /api/templates/{current}/inbox` 返回
**When** 渲染每条消息
**Then** 每项 `MessageItem`(高 72px,圆角 `rounded-sf`,`bg-shadowflow-surface`,`hover:bg-white/5`)包含:
- **左侧头像**(40×40 圆形,群聊用首字母 + `bg-shadowflow-accent`,单聊用 agent 首字母 + 对应颜色)
- **中部文字区**:
  - 第一行:`name`(font-semibold 14px)+ 右侧 `timestamp`(10px `text-white/50` mono)
  - 第二行:状态胶囊(`StatusPill`) + 最后消息预览(≤ 80 字符截断,`text-white/50 text-xs`)
- **右侧徽章区**(flex column,顶部对齐):
  - **未读 badge**:数字,> 99 显示 `99+`;圆形 `bg-shadowflow-accent text-white text-xs font-bold w-5 h-5 rounded-full`
  - **待决策徽章**(仅群聊):📋 N,当 N > 0 时显示 `text-shadowflow-warn`
**And** 无未读时不显示未读 badge(不占空间)

### AC3: StatusPill 组件

**Given** group 或 agent 有 `status` 字段
**When** 渲染 StatusPill
**Then** pill 样式:`rounded-full px-2 py-0.5 text-[10px] font-mono font-semibold`
- `running` → 背景 `#22C55E/15`,文字 `#22C55E`,文字 "Running"
- `blocked` → 背景 `#F59E0B/15`,文字 `#F59E0B`,文字 "Blocked"
- `idle` → 背景 `#6B7280/15`,文字 `#6B7280`,文字 "Idle"
- `pending_approval` → 背景 `#A78BFA/15`,文字 `#A78BFA`,文字 "Pending Approval"

### AC4: Tab 过滤(真实联动)

**Given** Story 7.1 建立的 4 Tab(全部 / 单聊 / 群聊 / 未读)
**When** 用户切换 Tab
**Then** 列表实时过滤:
- "全部":显示所有条目
- "单聊":仅 AGENT DMs section
- "群聊":仅 TEAM RUNS section
- "未读":仅 unreadCount > 0 的条目
**And** Tab 切换时 section header 根据内容动态显示/隐藏

### AC5: 数据接入

**Given** 前端调 `GET /api/templates/{current}/inbox`
**When** 页面加载或模板切换时
**Then** 返回:
```json
{
  "data": {
    "groups": [Group, ...],
    "agent_dms": [AgentDM, ...]
  },
  "meta": { "trace_id": "...", "timestamp": "..." }
}
```
- `Group` 含:`id / name / templateId / status / unreadCount / pendingApprovalsCount / lastMessage / lastActivityAt / metrics`
- `AgentDM` 含:`agentId / agentName / kind / status / unreadCount / lastMessage / lastActivityAt`

**Given** 新增 FastAPI endpoint `GET /api/templates/{template_id}/inbox`
**When** 前端请求
**Then** 从 `CheckpointStore`(已跑 runs)+ 内存 `AgentRegistry` 聚合返回实时数据
**And** P95 响应 ≤ 200ms

**Given** SSE 事件(`node.started` / `node.succeeded` / `policy.violation` / `approval.*`)到达
**When** 影响某群/agent 状态
**Then** `useInboxStore` 响应事件更新对应条目的 `status` / `unreadCount` / `pendingApprovalsCount`(不全量重拉)

### 关键约束

- **不借鉴钉钉 / 飞书 UI 用语**:字段名与 UI 文案统一用 ShadowFlow 术语(Group / Policy Matrix / ApprovalGate)
- **独立 store**:`useInboxStore` 严禁与 `useRunStore` 耦合。状态由 SSE 事件驱动增量更新,不全量 polling。
- **SSE 增量更新**:不在每次事件到达时全量重拉 `/api/templates/{id}/inbox`。事件驱动 `updateGroupStatus` action。
- **StatusPill 颜色 token**:与 Story 7.1 `tailwind.config.ts` `shadowflow.*` 颜色对齐。不硬编码 hex。

### 与 Story 7.4 边界

Story 7.2 负责渲染列表项(视觉 + 数据)。Story 7.4 负责点击后的导航逻辑(`onClick` 在 7.4 中实现)。本 Story 的 `MessageItem` 传 `onClick` prop 占位即可(console.log)。
