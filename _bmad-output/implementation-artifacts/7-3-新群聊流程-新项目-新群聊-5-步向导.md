# Story 7.3: "+ 新群聊"流程(新项目 = 新群聊 · 5 步向导)

Status: ready-for-dev
Created: 2026-04-21T07:05:42Z

---

## Story

As a **用户**,
I want **点击"+ 新群聊"一键开新项目**,
so that **我能快速把 AI 员工 + 人拉进同一个协作空间**。

---

## Acceptance Criteria

### AC1: 触发入口

**Given** Inbox 页 `MessageList` 顶部 Header 的"+ 新群聊"按钮(Story 7.1 占位 `console.log`)
**When** 用户点击
**Then** 弹出 `CreateGroupDialog`(全屏模态遮罩 + 居中卡片,宽 640px,圆角 `rounded-sf`,背景 `#161B22`)

### AC2: 5 步向导

**Given** Dialog 打开
**When** 渲染向导
**Then** 顶部步骤指示器(5 个圆点 + 连线,当前步高亮紫色 `#A78BFA`)

**Step 1 — 选 Group Template**:
- 列出当前模板 `groupRoster`(来自 `useInboxStore.currentTemplate.groupRoster`)
- 每项:group template name + agent 列表预览(3 个头像缩略 + "+N more")
- 单选,点击高亮选中(紫色边框)

**Step 2 — 选 AI 员工(Agent Roster)**:
- 来源:所选 group template 的 `agents` 列表(来自 `agentRoster`)
- 多选 checkbox,默认全选
- 每项显示 agent name + soul 简介(truncate 1 行)+ kind badge(ACP/CLI/MCP)

**Step 3 — 邀请人类成员(可选)**:
- 文本输入:email 或 user ID(换行分隔多个)
- 本 Story 仅前端 UI + 基础 email 格式验证(不做真实邀请发送,Phase 2 功能)
- 下方提示:"MVP:成员邀请 Phase 2 启用"(灰色小字)

**Step 4 — 命名群聊**:
- 文本输入,默认值 = 所选 group template name
- 最长 40 字符
- 实时字符计数

**Step 5 — 预览 Policy Matrix(可选微调)**:
- 继承所选 group template 的 `policyMatrix`
- 展示 N×N 矩阵(只读预览;若用户想微调,显示"进入群聊后可在 Policy Matrix 面板调整")
- `PolicyMatrixPanel` 只读模式(复用 Story 4.5 的展示层,非编辑层)

**And** 每步 [下一步] / [上一步] 按钮;步骤 5 [下一步] 变为 [创建群聊](紫色高亮按钮)

### AC3: 创建完成

**Given** 用户点击[创建群聊]
**When** 前端调 `POST /api/groups` body:`{ templateId, groupTemplateId, name, agentIds, memberEmails, policyMatrix }`
**Then** 返回 `{ groupId, name, ... }`
**And** 关闭 Dialog + Toast "✓ 群聊 '{name}' 已创建"
**And** 跳转到 `/chat/{groupId}`(Story 7.4 路由)
**And** `useInboxStore.groups` 追加新 group 条目(不全量重拉)

### AC4: 表单验证

**Given** 用户在各步操作
**When** 点击[下一步]
**Then** 当前步验证通过才能前进:
- Step 1:必须选一个 group template
- Step 4:群聊名不能为空
- 其他步骤可直接跳过(optional)

**And** 后端 `POST /api/groups` 返回 400 时:Toast 显示错误信息,Dialog 保持打开
